# Tabib — پلاتفۆرمی نۆرینگەی پزیشکی

Tabib is a medical appointment platform built with Flutter. It supports three roles in one codebase:

| Role | Features |
|------|----------|
| **Patient** | Register & login, search doctors by specialty, view profiles, book appointments, history, notifications |
| **Doctor** | Login, view appointments, accept/reject requests, patient records, write prescriptions |
| **Secretary** | Manage appointments, register patients, daily schedule |

**Stack:** Flutter · Material 3 · Clean Architecture · Firebase · go_router · Provider

**Languages:** Kurdish (Sorani) default RTL, Arabic RTL, English — switch via globe icon on any screen.

**Theme:** Blue, green, and white medical palette.

---

## Prerequisites

- Flutter 3.22+ (`flutter doctor`)
- Firebase project ([console.firebase.google.com](https://console.firebase.google.com))
- **Windows:** Enable [Developer Mode](https://learn.microsoft.com/en-us/windows/apps/get-started/enable-your-device-for-development)

## Firebase setup

### 1. Install FlutterFire CLI

```bash
dart pub global activate flutterfire_cli
```

### 2. Configure Firebase

```bash
flutterfire configure
```

This generates `lib/firebase_options.dart` and platform config files.

### 3. Enable services

1. **Authentication** — Anonymous (phone login) + Email/Password (staff & registered patients)
2. **Firestore** — Create database and apply security rules (see below)

### 4. Firestore indexes

Create composite indexes when prompted, or add manually:

| Collection | Fields |
|------------|--------|
| `appointments` | `patientId` Asc, `dateTime` Desc |
| `appointments` | `doctorId` Asc, `dateTime` Asc |
| `appointments` | `clinicId` Asc, `dateTime` Asc |
| `appointments` | `clinicId` Asc, `dateTime` Asc (range queries) |

### 5. Seed demo data

After admin login → **Seed demo data**, or call `AuthService.seedDemoData()` once.

### Example Firestore rules (development)

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

### Test accounts (create in Firebase Auth + Firestore `users` collection)

| Role | Method | Example |
|------|--------|---------|
| Patient | Phone name login | Any name + phone `07505555555` |
| Patient | Email register | Register screen |
| Doctor | Email/password | `doctor@clinic.app` + `role: doctor` in Firestore |
| Secretary | Email/password | `staff@clinic.app` + `role: secretary` in Firestore |

Firestore `users` document example for doctor:

```json
{
  "name": { "ku": "دکتۆر", "ar": "طبيب", "en": "Doctor" },
  "role": "doctor",
  "email": "doctor@clinic.app",
  "doctorId": "<doctor document id from doctors collection>"
}
```

## Project structure (Clean Architecture)

```
lib/
├── main.dart                 # Entry + Firebase bootstrap
├── app.dart                  # Root MaterialApp, providers, routing
├── firebase_options.dart     # Generated by flutterfire configure
│
├── core/
│   ├── constants/            # App-wide constants
│   ├── theme/                # Material 3 medical theme
│   ├── router/               # go_router routes & auth guards
│   ├── utils/                # Responsive breakpoints
│   └── widgets/              # Shared UI (logo, scaffold)
│
├── domain/
│   ├── entities/             # Domain entity exports
│   └── repositories/         # Repository interfaces
│
├── data/
│   └── repositories/         # Firestore repository implementations
│
├── models/                   # Data models (Appointment, Doctor, etc.)
│
├── presentation/
│   ├── providers/            # ChangeNotifier state (appointments, etc.)
│   └── screens/
│       ├── splash/
│       ├── auth/             # Login, Register
│       ├── patient/          # Home, doctors, booking, notifications
│       ├── doctor/           # Dashboard, prescriptions, records
│       └── secretary/        # Dashboard, register patient, schedule
│
├── services/                 # Auth, clinic data, locale, queue
│   └── backend/              # Firestore backend abstraction
│
├── l10n/                     # Kurdish, Arabic, English ARB files
└── widgets/                  # Reusable form & UI components
```

## Blockers

| Issue | Fix |
|-------|-----|
| Firebase not configured | Run `flutterfire configure` |
| Windows symlink error | Enable Developer Mode |
| Firestore index error | Create index from Firebase Console link |
| Anonymous auth disabled | Enable in Firebase Authentication |
